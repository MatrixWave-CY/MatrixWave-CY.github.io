标签页面

运行 hexo new page tags 命令，在MyBlog/source/会生成一个含有index.md文件的tags文件夹

标签页面（tag）在创建后，在表头加上type: "tags"和layout: "tags"，而内容留空

运行hexo new page categories 命令，在MyBlog/source/会生成一个含有index.md文件的categories文件夹

分类页面（categories） 在创建后，在**MyBlog/source/categories/index.md中**，添加type: "categories"

顶部图

顶部图的获取顺序，如果都没有配置，则不显示顶部图。

页面顶部图的获取顺序：各自配置的 top_img > 配置文件的 default_top_img

文章页顶部图的获取顺序：各自配置的 top_img > cover > 配置文件的 default_top_img

其它页面 （tags/categories/自建页面）和文章页的top_img，请到对应的 md 页面设置front-matter中的top_img

文章相关设置

  • 文章封面

    文章的markdown文档上，在Front-matter添加cover，并填上要显示的图片地址。如果不配置cover，可以设置显示默认的cover。

    如果不想在首页显示cover，可以设置为false。

  • 文章mete显示

  • 首页显示文章介绍

  • 页面锚点

    开启页面锚点后，当你在进行滚动时，页面链接会根据标题ID进行替换

    注意: 每替换一次，会留下一个歷史记录。所以如果一篇文章有很多锚点的话，网页的歷史记录会很多。

  • 图片描述

  • 文章内容复制相关配置

  • 文章目录TOC

    为特定的文章配置

    在你的文章md文件的头部，加入toc_number和toc，并配置true或者false即可。

    主题会优先判断文章 Markdown 的Front-matter是否有配置，如有，则以Front-matter的配置为准。否则，以主题配置文件中的配置为准。

  • 文章版权和协议许可

    如果有文章（例如：转载文章）不需要显示版权，可以在文章页Front-matter中单独设置

    支持对单独文章设置版权信息，可以在文章Front-matter单独设置

  • 赞助/打赏

  • 文章编辑

  • 相关文章推荐

  • 文章分页

  • 文章过期提醒

  • 文章置顶

    直接在文章的front-matter区域里添加sticky: 1属性来把这篇文章置顶。数值越大，置顶的优先级越大。

  • 主页文章节选

    自动节选和文章页description，**description在front-matter里添加**

  • 文章链接优化

    安装插件

    编辑 站点的 _config.yml 文件，找到 permalink , 改成 permalink: posts/:abbrlink/

    重新运行后插件会为每一篇文章生成唯一 id 并添加到 Front-matter

    由于 API 有 访问次数限制，为了避免调用太多，主题默认存取期限为 10 分鐘。也就是説，调用后资料会存在 localStorage 里，10分鐘内刷新网站只会去 localStorage 读取资料。 10 分鐘期限一过，刷新页面时才会去调取 API 读取新的数据
